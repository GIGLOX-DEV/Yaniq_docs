🎯 Authentication Service Example

@Service
public class AuthenticationService {
    private final CommonLogger logger = CommonLogger.getLogger(AuthenticationService.class);

    public LoginResponse authenticate(LoginRequest request) {
        String correlationId = UUID.randomUUID().toString().substring(0, 8);
        String maskedEmail = maskEmail(request.getEmail());

        logger.info("AUTH_START | CorrelationId: {} | Email: {}", correlationId, maskedEmail);

        try {
            // Authentication logic...
            logger.security("LOGIN_SUCCESS", correlationId, "User: {}", maskedEmail);
            return response;
        } catch (Exception e) {
            logger.security("LOGIN_FAILURE", correlationId, "User: {} | Error: {}",
                maskedEmail, e.getMessage());
            throw e;
        }
    }
}

⚙️ Configuration Guide

Environment Configuration

Configure CommonLogger using environment variables for different deployment scenarios.

Variable Default Description
LOG_MODE CONSOLE Logging strategy: CONSOLE, ELK, FILE, STRUCTURED
service.name yaniq-service Service name for ELK logs
yaniq.log.dir logs Log directory for file strategy
yaniq.log.debug false Enable debug level logging

🐳 Docker Configuration

version: '3.8'
services:
  auth-service:
    image: yaniq/auth-service:latest
    environment:
      - LOG_MODE=ELK
      - service.name=auth-service
      - service.version=1.2.0
      - yaniq.log.debug=false
    volumes:
      - ./logs:/var/logs/yaniq

☸️ Kubernetes Configuration

apiVersion: v1
kind: ConfigMap
metadata:
  name: logging-config
data:
  LOG_MODE: "ELK"
  service.name: "auth-service"
  service.version: "1.2.0"

💡 Best Practices

🔒 Security Best Practices

Never log sensitive information: passwords, credit cards, SSN, API keys

// ❌ BAD - Never do this
logger.info("User login: {} with password: {}", email, password);

// ✅ GOOD - Mask sensitive data
logger.info("User login: {}", maskEmail(email));

📊 Data Masking Utilities

private String maskEmail(String email) {
    if (email == null || email.length() < 3) return "***";
    int atIndex = email.indexOf('@');
    if (atIndex > 0) {
        String localPart = email.substring(0, atIndex);
        String domain = email.substring(atIndex);
        return localPart.substring(0, 2) + "***" + domain;
    }
    return email.substring(0, 2) + "***";
}

🎯 Correlation ID Usage

Always use correlation IDs for request tracking across microservices

public OrderResponse processOrder(OrderRequest request) {
    String correlationId = UUID.randomUUID().toString().substring(0, 8);

    logger.info("ORDER_START | CorrelationId: {} | CustomerId: {}",
        correlationId, request.getCustomerId());

    // All subsequent operations use the same correlation ID
    validateOrder(correlationId, request);
    processPayment(correlationId, request);

    logger.info("ORDER_SUCCESS | CorrelationId: {} | OrderId: {}",
        correlationId, order.getId());
}

🔄 Migration Guide

Step 1: Update Basic Logging Calls

// Before (v1.x)
logger.info("User " + userId + " authenticated in " + duration + "ms");

// After (v2.0)
logger.info("User {} authenticated in {}ms", userId, duration);

Step 2: Enhance with Structured Logging

// Before (v1.x)
logger.info("Authentication successful for user: " + user.getId());

// After (v2.0)
logger.info("AUTH_SUCCESS | CorrelationId: {} | UserId: {} | Duration: {}ms",
    correlationId, user.getId(), duration);

Step 3: Add Security and Audit Logging

// Before (v1.x)
logger.warn("Invalid password for user: " + user.getId());

// After (v2.0)
logger.security("LOGIN_FAILURE", correlationId,
    "User: {} | Reason: Invalid password", maskedEmail);
